CEN Studio Constitution v1.0

CEN Studio 개발 헌법

---

전문 (Preamble)

CEN Studio는 성경 데이터를 정확하고 신뢰성 있게 구축하여, 누구나 성경의 장소와 역사, 사건을 올바르게 탐험할 수 있도록 하는 것을 목적으로 한다.

우리는 화려한 기능보다 신뢰할 수 있는 데이터를 우선하며, 모든 개발은 장기적인 유지보수와 확장성을 고려하여 수행한다.

---

제1장. 핵심 가치

제1조 Data First

모든 기능보다 데이터의 정확성을 우선한다.

제2조 Place First

모든 탐색은 Place를 중심으로 이루어진다.

제3조 QA First

검증되지 않은 데이터는 Release하지 않는다.

제4조 Stable Before Smart

새로운 기술보다 안정성을 우선한다.

---

제2장. 데이터 원칙

제5조 Canonical DB

Canonical DB를 유일한 기준 데이터로 사용한다.

제6조 Runtime DB

Runtime DB는 Canonical DB로부터 생성된다.

제7조 JSON First

모든 운영 데이터는 JSON을 기본 형식으로 사용한다.

제8조 Expansion Rule

기존 Master DB를 직접 수정하지 않는다.
Expansion Batch를 통해 확장한다.

---

제3장. 개발 원칙

제9조 Batch Production

모든 작업은 Batch 단위로 수행한다.

제10조 Checkpoint

주요 마일스톤마다 GitHub Checkpoint를 생성한다.

제11조 Automation

반복 작업은 가능한 자동화한다.

제12조 QA Studio

QA Studio를 통해 데이터 품질을 지속적으로 검증한다.

---

제4장. BibleMaps 원칙

제13조 Direct Map

Direct Map이 존재하면 항상 최우선으로 사용한다.

제14조 Regional Map

Direct Map이 없을 경우에만 Regional Representative Map을 사용한다.

제15조 Era Map

Regional Map도 없을 경우에만 Era Representative Map을 사용한다.

제16조 User Trust

대표 지도를 개별 장소의 전용 지도처럼 표현하지 않는다.

---

제5장. Release 원칙

제17조 Release Rule

QA를 통과한 데이터만 Release한다.

제18조 Archive

모든 Release는 GitHub Archive에 보관한다.

제19조 Reproducibility

동일한 입력 데이터라면 동일한 결과를 생성할 수 있어야 한다.

---

제6장. 미래 비전

CEN Studio는 단순한 성경 앱이 아니라 성경 데이터 생산 플랫폼을 목표로 한다.

Places, Maps, Events, People, Keywords, Timeline, QA를 하나의 통합 개발 환경에서 관리한다.

---

CEN Studio Motto

Reliable Bible Data.
Built to Last.
Place First.
